## 1. 背景：这个实验在做什么？

### 1.1 现实中的问题

想象一根手指粗细的光纤，可以同时传输几百路高清视频通话。这靠的是**波分复用（WDM, Wavelength Division Multiplexing）**技术——把每路信号"染"成不同颜色（波长）的光，然后混合在一根光纤里一起传。到了接收端，再按颜色把它们分开，分别交给各自的接收器。

"按颜色分光"的器件叫**波长解复用器（Wavelength Demultiplexer）**。本项目设计的就是一种叫做**阵列波导光栅（AWG, Arrayed Waveguide Grating）**的芯片级解复用器——它把分光的功能做在一块指甲盖大小的硅芯片上。

### 1.2 为什么用 AWG？

早期的解复用器是用玻璃棱镜或衍射光栅做的，体积大、不易量产。AWG 的优势是：
- **集成度高**：整个器件做在硅芯片上，面积只有几平方毫米
- **CMOS 兼容**：可以用现有半导体工厂的工艺生产，成本低
- **性能稳定**：通道数多、插损低，是现代数据中心光互联的主流选择

### 1.3 本项目的具体目标

- **4个输出通道**，波长分别为 1547.6 / 1549.2 / 1550.8 / 1552.4 nm
- **通道间距**：1.6 nm（约 200 GHz），这是 DWDM（密集波分复用）的常见配置
- **工作波段**：C-band（约 1530–1565 nm），是光纤通信最核心的波段
- **仿真平台**：Tidy3D（云端电磁仿真）

## 1.4 整个项目的大白话通览（先读这里）

> 这一节不涉及任何公式和专业术语，只用日常语言把整个项目从头到尾讲一遍。建议先读这里，再去看后面的技术细节。

---

### 你可以把这个项目理解成：**用电脑设计一个"光的交通枢纽"**

想象你在一根头发丝粗细的玻璃管（光纤）里同时传输4路信号，每路信号用的是颜色略微不同的光——就像4台不同颜色的激光笔，红的、橙的、黄的、绿的，打进同一根管子里。到了另一头，你需要一个"分拣站"，把这4种颜色重新分开，各自送到对应的接收器。

**这个"分拣站"就是 AWG**，它做在一块比指甲还小的硅芯片上，里面雕刻着几十根比头发丝还细的光学通道。

---

### 第一步：先测量"管道"本身的性质

在设计这个分拣站之前，先要知道"光在这种硅管道里跑得有多快、感受到的折射率是多少"。这就像在修路之前，先测量一下地基的硬度。

于是用仿真软件（Tidy3D）画出一根 **500 纳米宽、220 纳米高的硅条**（比一根病毒还细），在里面"放"一束 1550 nm 的光，看它稳定下来的传播状态是什么样的。仿真告诉我们：

- 这根硅条能把光紧紧地箍住，让它只在里面跑，不外泄（只要波导够宽）
- 在 500 nm 宽度时，光感受到的等效折射率约是 **2.44**（比真空的1大，比硅本身的3.48小，这很合理，因为光有一部分跑到了外面的玻璃里）
- 换宽度，在400–600 nm 之间扫描了一遍，宽度越大折射率越大，规律清晰

这一步花了几分钱的云端计算费用，是整个项目中最便宜的部分。

---

### 第二步：用这个数据，算出分拣站的尺寸图纸

知道了折射率之后，就可以"纸上谈兵"地设计那个分拣站了。

AWG 分拣的原理其实很像乐队演奏：把16根长度不一样的波导排在一起，就像16根长短不同的风琴管。同一列光波进入这16根管子，在里面各自跑了不同的距离，出来的时候每根管子里的光的"节拍"（相位）就不一样了。当这16路光在出口处重新汇合，不同颜色的光因为"节拍"不同，汇聚的地方也不一样——这就实现了按颜色分光。

关键问题是：**相邻管子要比前一根长多少？** 算出来这个长度差是约 **31 微米**（比头发直径还小一点点）。16根管子里，最长的那根比最短的要长约 **465 微米**，大约是0.5毫米。

这一步完全靠公式推导，不花仿真费用。

---

### 第三步：用数学公式先粗略算算结果会是什么样

在真正做"电脑仿真实验"之前，先用一个简单的数学公式估算：如果这个分拣站完美运行，输出应该是什么样的光谱。

结论：4个输出通道的峰值波长正好在目标位置，插入损耗（光通过分拣站会损失多少功率）大约是 3 dB（即损失一半功率），通道间的串扰（不同颜色混在一起的程度）大约是 -13.7 dB。

**这个模型有个缺陷**：它是理想对称的，所以4个通道的数据完全相同。实际上，边缘的通道（颜色最偏的那两个）会更差一些。这只是个"预估值"，不是真实物理仿真。

---

### 第四步：用真正的电磁场仿真验证这个分拣站真的能工作

这才是整个实验的核心验证。

把芯片的一部分（光从管子出来、在自由空间里散射、再被收集到出口的那一段）放进仿真器里，让仿真器求解真实的麦克斯韦方程组——也就是让电脑真的去"模拟"光在里面的运动，就像用超高精度的相机拍下每一微秒里光是怎么传播的。

**花了多少计算费用？** 每次仿真约 0.07 个计算积分，总共跑了约20次，花了约 1.5 个积分（相当于完整仿真版本成本的约 1/100）。

**结论怎么样？** 仿真确实显示：
- 1547.6 nm 的光主要跑去了 3 号出口
- 1552.4 nm 的光主要跑去了 4 号出口

波长越长，光越往一侧偏。这个现象是真实物理仿真的结果，不是数学公式推出来的，证明了这个分拣站**确实有分光能力**。

但分得不够干净——大概只有 33–39% 的功率准确进入了"正确"的出口，其他功率散在了别的出口里。这就是为什么分拣站还需要优化。

---

### 第五步：改改参数，让分拣站分得更干净

找到了问题之后，就开始系统地调整参数。可以调的参数主要有三个：

- **管子之间的间距**（阵列间距）：间距太大，光出来散得太开；太小，相邻管子里的光会互相干扰。
- **出口之间的间距**（输出间距）：出口要正好对准光汇聚的位置，太宽则漏光，太窄则串扰。
- **出口阵列的整体偏移**（输出偏移）：相当于微调出口位置，让最强的光刚好打在出口中心。

**第一轮**：先用一个超便宜的本地近似模型扫描了 375 种参数组合，找出最有希望的方向（不花云端费用）。结论是：间距应该从初始的 1.5 μm 收紧到约 1.2 μm。

**第二轮**：对约9个候选参数组合跑真实 FDTD 仿真，只看中间那个波长（1550.8 nm）。找到最优参数后，中间波长的分光效果从 33% 提升到了 47%，质量从"合格"升到了"良"。

**第三轮**：把优化目标改成"4个波长平均最好"，而不是只看一个。结论是：第二轮找到的那组参数，恰好也是4个波长平均最好的。只是整体平均只提升了约 5%，4个通道还没有全部达到"良"的水平。

---

### 第六步：改变"管子数量"，让分光更干净

前面几步的优化都只调整了出口位置和间距，但有一个参数一直没变——**阵列臂的数量**（始终是 16 根）。

从物理上说，16 根管子形成的干涉图样，峰值宽度约为 1.2 nm，和 1.6 nm 的通道间距太接近了，相邻通道的光互相"漏"进去。就像用 16 条缝的光栅，分辨率不够高。

把管子从 16 根增加到 24 根之后：
- 干涉峰宽变为 0.8 nm，明显窄于 1.6 nm 的间距
- 同时把出口间距从 1.6 μm 调整到 1.8 μm，给每个通道更多"停车位"

结果：**4个通道里有 2 个达到了"良（general）"级别**，全通道平均分光效果提升了约 15%。这是整个优化过程中单次改动效果最显著的一步。

剩余的问题（1550.8 nm 通道还是只有"合格"级别）需要从根本上改变出口的几何排列方式（用"Rowland 圆"替代直线排列）才能解决，这超出了当前简化模型的能力范围。

---

### 最终结论（一句话）

**成功设计并仿真了一个4通道硅基波长分拣站，通过 4 轮共 ~36 次云端 FDTD 仿真（总计约 3.2 FlexCredit）证明了器件按波长分光的能力，最终有 2 个通道达到"良"级，全通道平均分光效率从初始的 34.9% 提升至 40.3%，约为全精度仿真成本的 1/100。**

下一步若要进入实际流片，需要做完整3D仿真（约贵100倍）和 Rowland 圆几何优化。

## 2. 光波导的物理基础

### 2.1 为什么光会被"关"在波导里？

核心物理是**全内反射（Total Internal Reflection, TIR）**。当光从折射率高的材料射向折射率低的材料，且入射角足够大时，光会被完全反射回来，一点都不透出去。

本项目用的**硅条形波导（Silicon Strip Waveguide）**截面示意：

```
     ←── 宽度 w（本项目：500 nm）──→
     ┌──────────────────────────────┐   ↑
     │  核心：硅（Si）  n = 3.48     │  高度 h = 220 nm
     └──────────────────────────────┘   ↓
─────────────────────────────────────────── SiO₂ 衬底，n = 1.44
         上下左右均为 SiO₂ 包层，n = 1.44
```

- 硅的折射率（3.48）远大于 SiO₂（1.44），折射率差非常大
- 光进入硅核心后，在所有界面都发生全内反射，被紧紧限制在 500 nm × 220 nm 的截面内沿轴向传播
- 这个尺度（几百纳米）与光的波长（约 1550 nm）相当，所以必须用电磁场理论来分析，而不能用简单的几何光学

### 2.2 什么是"模式"（Mode）？

不是所有形状的波都能在波导里稳定传播。只有满足特定边界条件的电磁场分布，才能在波导里持续传播而不发散，这些特定的场分布叫做**导模（Guided Mode）**。

可以用一个形象的比喻：波导就像一根弦乐器的琴弦。弦上能稳定振动的，只有满足驻波条件的那几个频率（基频、2倍频、3倍频……）。光在波导里也一样，只有满足边界条件的场分布才能存在。

- **基模（Fundamental Mode, mode 0）**：能量最集中在核心中央，传播最稳定。本项目就工作在基模
- **高阶模（Higher-order Mode）**：场分布更复杂，有多个峰。如果波导太宽，会同时支持多个模，造成干扰

对于 500 nm × 220 nm 的硅波导，在 1550 nm 波长下，只支持一个 TE 方向的基模（**准单模工作**），这对 AWG 设计很重要——如果有多模混叠，信号就会串扰。

### 2.3 传播常数（Propagation Constant）β

光在波导里沿轴向（z方向）传播，其电场可以写成：

$$E(z, t) = E_0 \cdot e^{j(\beta z - \omega t)}$$

这里 β 就是**传播常数**，单位是 rad/μm（每微米积累的相位弧度数）。它的定义是：

$$\beta = \frac{2\pi}{\lambda} \cdot n_{\text{eff}}$$

其中 λ 是真空中的光波长，$n_{\text{eff}}$ 是有效折射率。这个公式的物理意义是：光传播 1 个波长的距离，在真空中积累 2π 的相位；但在波导里因为折射率更高，等效波长变短，所以同样的物理距离会积累更多相位。

### 2.4 有效折射率 n_eff 是什么？怎么来的？

**大白话**：n_eff 是光"感受到"的折射率。光场的大部分能量在 Si 里（n=3.48），但有一部分"渗透"到了 SiO₂（n=1.44）里，所以实际感受到的折射率是两者的加权平均，但不是简单算术平均，而是由麦克斯韦方程组的解决定。

**精确定义**：n_eff 是使麦克斯韦方程组的本征方程有解的那个特征值。数学上，求解波导模式等价于求解一个本征值问题，n_eff 就是本征值，对应的电磁场分布就是本征函数（即"模式"）。

Tidy3D MODE 求解器做的事就是：给定波导截面的几何和材料，用数值方法解这个本征值问题，返回每个模式的 n_eff 和场分布。

**直觉**：
- 波导变宽 → 光被束缚得更好，在 Si 里的比例更大 → n_eff 更接近 Si 的折射率（3.48），所以**越宽 n_eff 越大**
- 波导变窄 → 光往外泄漏更多，在 SiO₂ 里的比例更大 → n_eff 更接近 SiO₂（1.44），所以**越窄 n_eff 越小**

### 2.5 群折射率 n_group 是什么？为什么要区分它？

**大白话**：n_eff 告诉你"波峰以多快速度移动"（相速度），n_group 告诉你"光脉冲的包络以多快速度移动"（群速度）。通信传输的是脉冲（0和1），所以我们关心的是群速度，因此 AWG 设计里用的是 n_group。

**数学关系**：

$$n_g = n_{\text{eff}} - \lambda \frac{dn_{\text{eff}}}{d\lambda}$$

第二项 $\lambda \frac{dn_{\text{eff}}}{d\lambda}$ 反映了**色散（Dispersion）**：不同波长的光在波导里传播速度不同。对于硅波导，dn_eff/dλ < 0（波长越长，n_eff 越小），所以 $-\lambda \frac{dn_{\text{eff}}}{d\lambda} > 0$，这意味着 n_group > n_eff，群速度比相速度慢。

**为什么 AWG 设计用 n_group 而不是 n_eff？**

AWG 的工作原理依赖于"不同波长的光在相同长度的波导里积累不同的相位"。当波长变化 Δλ 时，相位的变化量是：

$$\Delta\phi = \frac{2\pi}{\lambda^2} \cdot n_g \cdot L \cdot \Delta\lambda$$

这里出现的是 n_group，不是 n_eff。直觉上理解：相位对波长的敏感度，由色散特性决定，而色散正是 n_group 和 n_eff 之差的体现。

**数值结果**（500 nm × 220 nm，1550 nm）：
- n_eff = 2.4446（相速度折射率）
- n_group = 4.0512（群速度折射率，比 n_eff 大约 66%）

## 3. AWG 的工作原理与核心公式推导

### 3.1 AWG 的五个组成部分

```
                                       臂0: 基准长度 L₀
                                       臂1: L₀ + ΔL
    输入光                              臂2: L₀ + 2ΔL
    (混合了所有      ┌──FPR1──┐          ···                 ┌──FPR2──┐    → ch1: 1547.6 nm
     λ₁~λ₄)  ─────►│自由传播区│──────────────────────────────►│自由传播区│    → ch2: 1549.2 nm
                   │（星形   │         臂15: L₀ + 15ΔL      │（星形   │    → ch3: 1550.8 nm
                   │ 耦合器）│                               │ 耦合器）│    → ch4: 1552.4 nm
                   └─────────┘                               └─────────┘
```

1. **输入波导**：携带混合了4路波长信号的光进入器件
2. **输入 FPR（自由传播区/星形耦合器）**：光在这个平板波导里自由扩散，均匀地耦合进 N 根阵列臂。扩散的物理原因是**衍射（Diffraction）**——光到达 FPR 后不再受到横向约束，像水波一样向四周扩散
3. **阵列波导（Arrayed Waveguides）**：16 根波导，长度依次递增，相邻臂之间长度差恒为 ΔL。光在每根臂里独立传播，各自积累相位
4. **输出 FPR**：16 路光从阵列臂射出，在第二个自由传播区里重新汇聚。由于每根臂携带着不同的相位，汇聚时发生**干涉**，不同波长的光因为相位模式不同，汇聚到不同的横向位置
5. **输出波导**：放在 FPR 输出面的不同横向位置，每个位置对应一个波长通道

### 3.2 AWG 分光的本质：相控干涉

AWG 的分光原理和**相控阵雷达（Phased Array Radar）**完全类似。理解相控阵就理解了 AWG。

**相控阵的原理**：在一排天线阵列中，给每根天线一个略微不同的延迟。这样，不同方向上的信号叠加时，有的方向相长干涉（信号加强），有的方向相消干涉（信号消失）。通过调整各天线的延迟，可以把波束"指向"特定方向。

**AWG 里发生了什么**：16 根阵列臂就是 16 根"天线"。每根臂的长度差 ΔL 给它们引入了固定的额外相位延迟。当这 16 路光在输出 FPR 里干涉时：
- 对于某个特定波长 λ₁，这些延迟恰好让干涉极大值出现在输出面的位置 y₁
- 对于另一个波长 λ₂（稍长），干涉极大值出现在另一个位置 y₂

这就是 AWG "按波长分光"的本质。

### 3.3 核心公式推导：从第一原理到 ΔL

**第一步：单根臂内的相位积累**

光通过第 k 根阵列臂（额外长度 k·ΔL）积累的相位：

$$\phi_k(\lambda) = \beta(\lambda) \cdot (L_0 + k \cdot \Delta L) = \frac{2\pi \cdot n_{\text{eff}}(\lambda)}{\lambda} \cdot (L_0 + k \cdot \Delta L)$$

**第二步：干涉条件**

在输出 FPR 的中心位置（y=0，对应"0级衍射"）发生相长干涉的条件是：相邻臂之间的相位差必须是 2π 的整数倍：

$$\phi_{k+1}(\lambda_0) - \phi_k(\lambda_0) = \frac{2\pi \cdot n_{\text{eff}}(\lambda_0)}{\lambda_0} \cdot \Delta L = 2\pi m$$

整理得：

$$\boxed{\Delta L = \frac{m \cdot \lambda_0}{n_{\text{eff}}(\lambda_0)}}$$

其中 m 是正整数，叫做**衍射级次（Diffraction Order）**。

> 注：这里先写成 n_eff 形式，下一步会解释为什么实际计算用 n_group。

**第三步：为什么 ΔL 公式里要用 n_group？**

上面的公式是针对单一波长 λ₀ 的相长干涉。但 AWG 的分光能力来自于"当波长偏离 λ₀ 时，干涉极大值的横向位置如何移动"。

设波长偏移量为 Δλ = λ - λ₀，干涉极大值的横向位移 Δy 正比于：

$$\Delta y \propto \frac{\partial \phi}{\partial \lambda} \cdot \Delta\lambda = \frac{\partial}{\partial \lambda}\left(\frac{2\pi \cdot n_{\text{eff}}(\lambda)}{\lambda}\right) \cdot \Delta L \cdot \Delta\lambda$$

对括号内求导：

$$\frac{\partial}{\partial \lambda}\left(\frac{n_{\text{eff}}}{\lambda}\right) = \frac{1}{\lambda}\frac{dn_{\text{eff}}}{d\lambda} - \frac{n_{\text{eff}}}{\lambda^2} = -\frac{1}{\lambda^2}\underbrace{\left(n_{\text{eff}} - \lambda\frac{dn_{\text{eff}}}{d\lambda}\right)}_{= n_g}$$

所以干涉极大值的移动速度正比于 **n_group**，而不是 n_eff。因此，在计算给定通道间距需要多少 ΔL 时，公式变为：

$$\boxed{\Delta L = \frac{m \cdot \lambda_0}{n_g}}$$

**大白话**：n_eff 决定的是在某个波长下相长干涉在哪里；n_group 决定的是当波长变化时，干涉极大值的位置变化有多快。AWG 分光靠的是后者，所以用 n_group。

### 3.4 FSR（自由光谱范围）

AWG 是"周期性"的器件——当波长增加一个 FSR，干涉条件重复，极大值回到原来的位置（但这已经是下一个衍射级了）。超出 FSR 范围的信号会重叠到其他通道上，产生混叠。

**推导**：FSR 是让相位差从 2πm 变成 2π(m+1) 所需要的波长变化量。由 m·λ₀ = n_g·ΔL，当 m 不变而 λ 变化 FSR 时：

$$m \cdot (\lambda_0 + \text{FSR}) = n_g \cdot \Delta L + n_g \cdot \Delta L \cdot \frac{\text{FSR}}{\lambda_0} \approx n_g \cdot \Delta L$$

（忽略色散的二阶项后）化简得：

$$\boxed{\text{FSR} = \frac{\lambda_0}{m}}$$

**直觉**：衍射级次 m 越大，FSR 越小。m 大意味着 ΔL 长，说明阵列臂对波长更敏感，极大值移动得更快，所以 FSR（一个完整周期对应的波长范围）就更小。

### 3.5 通道位置的推导（Grating Equation）

波长为 λ 的光在输出 FPR 输出面的横向聚焦位置 y，由以下方程决定（类比衍射光栅方程）：

$$n_s \cdot d \cdot \sin\theta_i + n_s \cdot d \cdot \sin\theta_o = m \cdot \lambda$$

其中 d 是阵列间距（array pitch），$\theta_i$ 是输入侧的衍射角，$\theta_o$ 是输出侧的聚焦角，$n_s$ 是 FPR 平板波导的折射率。

**直觉**：波长增大 → 为满足等式，$\sin\theta_o$ 必须增大（当 $\theta_i$ 固定时） → 输出焦点向外侧移动。这就是为什么长波长通道对应的输出端口位置更靠外。

### 3.6 本项目的数值计算

将 λ₀ = 1550 nm，n_g = 4.0512，FSR 目标 = 19.2 nm 代入：

**步骤1：确定衍射级次 m**

$$m = \frac{\lambda_0}{\text{FSR}} = \frac{1550}{19.2} \approx 80.7 \rightarrow \text{取整为 } m = 81$$

实际 FSR：$\frac{1550}{81} \approx 19.136 \text{ nm}$，比4通道总跨度4.8 nm 大约4倍，安全裕量充足。

**步骤2：计算路径长度差 ΔL**

$$\Delta L = \frac{m \cdot \lambda_0}{n_g} = \frac{81 \times 1550 \text{ nm}}{4.0512} \approx 30.99 \text{ μm}$$

## 4. 仿真工具与策略

### 4.1 Tidy3D 是什么？

Tidy3D 是一个基于**时域有限差分法（FDTD, Finite-Difference Time-Domain）**的云端电磁仿真平台。

**FDTD 的本质**：把空间划分成一个网格（Yee Grid），然后在时间上一步一步地更新每个网格点上的电场 E 和磁场 H，依据麦克斯韦方程组的旋度方程：

$$\nabla \times \mathbf{E} = -\mu_0 \frac{\partial \mathbf{H}}{\partial t}, \quad \nabla \times \mathbf{H} = \epsilon \frac{\partial \mathbf{E}}{\partial t}$$

初始时注入一个宽带脉冲光源，让它在结构里传播，记录各处的时域响应，最后对时域数据做**傅里叶变换（FFT）**得到频域（即波长域）的传输谱。

**FDTD 的主要成本来自哪里？**

成本 ∝ 网格点数 × 时间步数

网格分辨率通常需要达到波长的 1/20 以上才能保证精度。在 1550 nm 波长下，网格间距约 1550/20 ≈ 77 nm。一个 1000 μm × 100 μm × 0.5 μm 的区域，需要约 1.3×10⁸ 个网格点，每次时间步都要更新所有点。这就是为什么大规模 FDTD 需要大量算力（GPU 集群），以及为什么本项目尽量避免运行完整 3D FDTD。

**MODE Solver 是什么？**

MODE 求解器不跑时域演化，而是直接求解稳态的本征方程：给定波导截面，找到使麦克斯韦方程有周期解的 n_eff 值。计算量远小于 FDTD，成本仅约 0.006 FlexCredit 每次，约是 FDTD 的 1/1000。

### 4.2 FlexCredit（计算积分）

Tidy3D 用 FlexCredit 计量计算资源。本项目的成本结构：

| 阶段 | 每次成本 | 运行次数 | 小计 |
|------|---------|---------|------|
| MODE 宽度扫描（5次） | ~0.006 | 5 | ~0.03 |
| 2D FDTD 验证（含优化） | ~0.074 | ~20 | ~1.5 |
| **本地阵列因子模型** | **0** | - | **0** |
| 完整 3D FDTD（**未做**） | ~7.8 | - | - |
| **总计** | - | - | **~1.5** |

### 4.3 三层仿真策略的逻辑

本项目的核心策略是**用最低的成本，逐步建立对器件物理行为的信心**。三个层次分别回答三个问题：

```
层次 1: MODE（~0.006/次）
   问题："这根波导里光的基本参数是什么？"
   输出：n_eff、n_group，作为后续计算的输入
   
层次 2: 本地阵列因子模型（免费）
   问题："如果只有理想干涉，传输谱长什么样？"
   输出：理想化的插损、串扰、带宽预测
   局限：不考虑真实几何细节和模式耦合
   
层次 3: 2D FDTD（~0.074/次）
   问题："在真实电磁场传播下，光确实被路由到了正确的端口吗？"
   输出：各端口的真实功率分布，验证分光效果
   局限：用 2D 有效折射率近似替代了真实 3D 截面
   
（未做）层次 4: 完整 3D FDTD（~7.8/次）
   问题："完整器件的准确插损和串扰是多少？"
   这是性能评估的金标准，也是课程时间/预算的瓶颈
```

## 5. 第一阶段：波导 MODE 分析

### 5.1 仿真设置

在 Tidy3D 里建立一个 2D 截面模型：

- **核心**：矩形 Si 区域，宽度 w（扫描参数），高度 h = 220 nm，折射率 n = 3.48
- **包层**：四周全部为 SiO₂，折射率 n = 1.44
- **求解目标**：找到 TE-like 基模（电场主要分量平行于芯片平面的最低阶模）的 n_eff

220 nm 高度是**固定的工艺参数**，代表 SOI（Silicon-on-Insulator，绝缘体上硅）晶圆的器件层厚度。这是业界最常见的平台，fab 工厂能做的最小特征尺寸和该厚度高度匹配。

### 5.2 宽度对折射率的影响

**直觉解释**：想象波导截面就像一个"光的容器"。容器越宽，光越不容易"溢出"到包层里，被束缚在高折射率区域的比例越大，所以 n_eff 越大。但同时，宽波导的色散也更弱（波导色散减小），所以 n_group - n_eff 这个差值（由色散决定）随宽度增大而减小，体现为 n_group 随宽度增大而减小。

**宽度扫描完整结果（在 1550 nm 波长）：**

| 宽度（nm） | n_eff | n_group | n_group - n_eff（色散贡献） |
|-----------|-------|---------|--------------------------|
| 400 | 2.2275 | 4.2449 | +2.017 |
| 450 | 2.3527 | 4.1444 | +1.792 |
| **500** ★ | **2.4446** | **4.0512** | +1.607 |
| 550 | 2.5131 | 3.9749 | +1.462 |
| 600 | 2.5657 | 3.9142 | +1.349 |

★ 选定设计基准宽度

**规律总结**：
- n_eff 单调增大（波导越宽，束缚越好，n_eff 越接近 Si 的 3.48）
- n_group 单调减小（波导越宽，波导色散越弱，n_group 越接近 n_eff）
- n_group - n_eff 的差值减小，说明宽波导的色散较弱

### 5.3 为什么选 500 nm？

三个理由：

1. **跟 baseline 一致**：项目最初的 baseline 仿真就是 500 nm，换一个宽度意味着要重新校准所有参数
2. **处于中间区间**：n_eff 和 n_group 都不极端，设计裕量好
3. **工程实践验证**：硅光子领域，500 nm × 220 nm 是经过无数流片验证的"最安全"尺寸，在这个尺寸下波导基本单模，弯曲损耗也可控

## 6. 第二阶段：AWG 初始参数设计

### 6.1 通道规划

4个通道对称分布在 1550 nm 两侧，间距均为 1.6 nm：

| 通道 | 中心波长（nm） | 中心频率（THz） | 与中心波长的偏移 |
|------|--------------|----------------|----------------|
| 1    | 1547.6 | 193.714 | -2.4 nm |
| 2    | 1549.2 | 193.514 | -0.8 nm |
| 3    | 1550.8 | 193.315 | +0.8 nm |
| 4    | 1552.4 | 193.116 | +2.4 nm |

- **为什么不直接以 1550 nm 为一个通道中心？** 对称设计让中心波长 1550 nm 恰好落在通道 2 和通道 3 之间，这样 AWG 的"中心对称"设计点就是器件的数学中心，而不是某个通道的峰值位置，更稳定
- **1.6 nm ≈ 200 GHz** 的转换：频率间距 Δf ≈ c × Δλ / λ² = 3×10⁸ m/s × 1.6×10⁻⁹ m / (1550×10⁻⁹ m)² ≈ 200 GHz

### 6.2 FSR 的选择逻辑

**必须满足的条件**：FSR 要远大于4通道的总跨度（4.8 nm），否则其他衍射级的光会叠进通道里。

**设计选择**：FSR ≈ 4 × 4.8 nm = 19.2 nm（4倍的跨度）

**裕量说明**：选 4 倍是工程经验值。越大越安全，但 m 越小意味着 ΔL 越小，臂长差异越小，精度要求越高，同时频谱选择性也越低。4倍是合理折衷。

### 6.3 参数推导链：从 FSR 到 ΔL 的完整计算

```
已知：
  λ₀ = 1550 nm
  n_eff = 2.4446
  n_g = 4.0512
  目标 FSR = 19.2 nm

Step 1：求衍射级次 m
  m = λ₀ / FSR_target = 1550 / 19.2 = 80.7 → 取整 m = 81
  实际 FSR = 1550 / 81 = 19.136 nm ✓（比 4.8 nm 大约 4 倍，合格）

Step 2：求路径长度差 ΔL
  ΔL = m × λ₀ / n_g
     = 81 × 1.550 μm / 4.0512
     = 125.55 / 4.0512
     = 30.991 μm

Step 3：验证物理意义
  在 1550 nm，臂间相位差 = 2π × n_eff × ΔL / λ₀
                          = 2π × 2.4446 × 30.991 / 1.550
                          = 2π × 81.0 ≈ 2π × 81 ✓
  （正好是 81 个完整周期，符合 m=81 的设计）
```

| 参数 | 初始值 | 物理含义 |

## 7. 第三阶段 Part A：本地阵列因子频谱模型

### 7.1 为什么要做这个"免费"模型？

直接跑 FDTD 验证需要花 FlexCredit，而且每次改参数都要重新跑。在进入 FDTD 之前，先用一个纯数学模型预测一下"如果一切理想，传输谱大概是什么形状"，可以帮助检查设计逻辑是否有明显错误，而且不花一分钱。

### 7.2 阵列因子（Array Factor）公式及推导

设 N 根等间距阵列臂，相邻臂之间的路径长度差为 ΔL。对于偏离设计中心波长 λ₀ 的波长 λ = λ₀ + δλ，相邻臂之间的相位差为：

$$\Delta\phi(\lambda) = \frac{2\pi \cdot n_g}{\lambda^2} \cdot \Delta L \cdot \delta\lambda \approx \frac{2\pi}{\text{FSR}} \cdot \delta\lambda$$

（最后一步用了 $\Delta L = m\lambda_0/n_g$，FSR = $\lambda_0/m$）

所以相邻臂的相位差就是 $\phi = 2\pi \delta\lambda / \text{FSR}$，N 根臂叠加的总场振幅（类比天线阵列）：

$$A(\delta\lambda) = \sum_{k=0}^{N-1} e^{j k \phi} = \frac{\sin(N\phi/2)}{\sin(\phi/2)} \cdot e^{j(N-1)\phi/2}$$

功率正比于振幅的平方，即**阵列因子（Array Factor）**：

$$\text{AF}(\delta\lambda) = \left|\frac{\sin(N\phi/2)}{N\sin(\phi/2)}\right|^2, \quad \phi = \frac{2\pi \cdot \delta\lambda}{\text{FSR}}$$

**这个公式的物理含义**：
- 当 δλ = 0（即 λ = λ₀，中心通道）：$\phi = 0$，分子分母都趋向于 0，用洛必达法则得 AF = 1，全部光汇聚在中心
- 当 δλ = FSR：$\phi = 2\pi$，AF 再次 = 1，这是下一个衍射级
- 当 δλ = FSR/N：$\phi = 2\pi/N$，这是第一个零点附近，光场趋向抵消
- N 越大，主峰越尖锐，旁瓣越多（类比天线方向图）

### 7.3 高斯包络（Gaussian Envelope）的来源

即使在理想阵列因子的基础上，实际传输还要乘以一个包络，原因是：每根阵列臂的出口处，光的场分布是高斯形状的，在 FPR 输出面上与输出波导模式的重叠积分（Overlap Integral）随离轴距离增大而减小。近似地，这个包络可以用高斯函数建模：

$$G(\delta\lambda) = \exp\left(-\frac{\delta\lambda^2}{2\sigma^2}\right), \quad \sigma = 6 \text{ nm（经验参数）}$$

### 7.4 完整传输谱公式

$$T(\lambda) = T_{\text{peak}} \cdot \underbrace{\left|\frac{\sin(N\phi/2)}{N\sin(\phi/2)}\right|^2}_{\text{阵列因子：干涉图样}} \cdot \underbrace{\exp\!\left(-\frac{\delta\lambda^2}{2\sigma^2}\right)}_{\text{高斯包络：耦合效率}}$$

其中 $T_{\text{peak}} = 10^{-3.0/10} \approx 0.501$（对应 3 dB 的峰值插入损耗）

**为什么峰值不是 1（0 dB），而是 0.5（-3 dB）？**

即使在理想情况下，一个输入口的光进入 FPR 后是散射到所有方向的，而不是所有能量都进入某一个特定输出口。能量在各输出口之间按照阵列因子图样分配。对于简化版课程模型，先手动将峰值设定为 -3 dB（即 50% 的功率进入目标通道），这是一个保守但合理的工程估计。

### 7.5 模型计算结果

| 通道 | 目标波长（nm） | 峰值波长（nm） | 插入损耗（dB） | 串扰（dB） | 3dB带宽（nm） |
|------|--------------|--------------|----------------|------------|---------------|
| 1 | 1547.6 | 1547.6 | 3.00 | -13.71 | 1.060 |
| 2 | 1549.2 | 1549.2 | 3.00 | -13.71 | 1.060 |
| 3 | 1550.8 | 1550.8 | 3.00 | -13.71 | 1.060 |
| 4 | 1552.4 | 1552.4 | 3.00 | -13.71 | 1.060 |

**重要局限：** 这4组数字完全相同，因为模型是对称的——它把4个通道都看作"偏离中心 ±0.8 nm 或 ±2.4 nm"的情况，对称性相同所以结果一样。**真实器件中边缘通道（ch1 和 ch4）会有更大的插损**，因为它们距离 FPR 的几何中心更远，耦合效率更低。

## 8. 第三阶段 Part B：2D FDTD 高保真器件验证

### 8.1 为什么不直接跑完整 3D FDTD？

完整 AWG 器件的尺寸：阵列臂区域约 2 mm 长，FPR 约 50 μm，加上波导弯曲，总尺寸约 2 mm × 0.3 mm。

3D FDTD 需要对整个体积做网格划分：
$$\text{网格点数} \approx \frac{2000\,\mu\text{m} \times 300\,\mu\text{m} \times 1\,\mu\text{m}}{(0.08\,\mu\text{m})^3} \approx 1.2 \times 10^{10}$$

这已经超出了普通云端仿真的内存上限，且成本约 7.8 FlexCredit，接近本项目全部计算预算的 5 倍。

### 8.2 2D 有效折射率方法：如何降维？

**核心思路**：把三维问题变成二维问题，同时保留最关键的物理——光在芯片平面内的传播和干涉。

**具体做法**：

1. **放弃 z 方向（厚度方向）的分辨率**：不再在 220 nm 高度方向上细分网格，而是用 n_eff（已从 MODE 求解器得到）来等效这个方向的作用。这相当于把 3D 波导"压扁"成 2D 平面，但在这个 2D 平面里，材料折射率用 n_eff 替代。

2. **用预设相位代替实际阵列臂**：不在 FDTD 里真正建模 16 根蜿蜒 2 mm 的阵列波导，而是在阵列臂出口处，直接设置对应波长和臂编号的相位值：
   $$\phi_k(\lambda) = \frac{2\pi \cdot n_{\text{eff}}(\lambda)}{\lambda} \cdot k \cdot \Delta L$$
   然后用这组相位值构造 16 个点源，注入 FPR 进行传播求解。

3. **只对输出 FPR 区域做精细 FDTD**：这是最关键的部分——光从阵列臂出来后，在 FPR 里的传播和在输出波导里的聚焦，才是决定分光质量的核心。这个区域的大小约 60 μm × 30 μm，2D FDTD 成本约 0.074 FlexCredit。

**这个近似保留了什么物理？**
- ✅ 相控阵干涉（不同波长产生不同的输出焦点位置）
- ✅ FPR 内的衍射传播
- ✅ 输出波导的模式耦合

**这个近似忽略了什么？**
- ❌ 阵列臂弯曲处的辐射损耗
- ❌ 阵列臂之间的模式耦合（当臂间距很小时）
- ❌ 3D 模式的横向泄漏
- ❌ 制造误差（波导宽度不均匀等）

### 8.3 评价指标的定义

为了量化 FDTD 仿真的分光质量，定义了两个指标：

**指标1：主输出占比（Dominant Fraction）**

$$\text{Dominant Fraction} = \frac{P_{\text{dominant}}}{\sum_{i=1}^{4} P_i}$$

其中 $P_i$ 是第 i 个输出端口的检测到的总功率，$P_{\text{dominant}}$ 是4个输出中最大的一个。

**大白话**：如果你把4个输出口总功率当做 100%，有多少比例留在了最强的那个端口。理想 AWG 应该是 100%（所有功率进入目标通道），实际器件由于串扰，这个比例会低于 100%。

**指标2：次强/最强比（Second/First）**

$$\text{Second/First} = \frac{P_{\text{2nd strongest}}}{P_{\text{strongest}}}$$

**大白话**：第二强的输出端口的功率是最强端口功率的几倍。这个值越小越好——越小说明主输出和次强输出之间差距越大，分光越干净。

### 8.4 质量评级标准（Routing Quality Rubric）

| 等级 | Dominant Fraction | Second/First | 实际含义 |
|------|------------------|--------------|---------|
| **Good（优）** | ≥ 0.60 | ≤ 0.50 | 60% 以上功率进入主通道，次通道功率不到主通道一半。这才是真正好用的解复用器 |
| **General（良）** | ≥ 0.40 | ≤ 0.80 | 至少40%进入主通道。功能上可用，但串扰还比较高 |
| **Passing（合格）** | ≥ 0.30 | ≤ 0.95 | 至少能看出波长路由的方向，但分离很不干净。勉强合格，证明原理 |
| **Poor（差）** | < 0.30 或 > 0.95 | 超限 | 几乎没有分光效果，近似于均匀分光 |

### 8.5 Baseline FDTD 验证结果

**初始几何参数**：array pitch = 1.5 μm，output pitch = 2.0 μm，output offset = 0.0 μm

在4个目标波长下运行 2D FDTD，记录各输出端口的功率：

| 波长（nm） | 主输出 | out_1 | out_2 | out_3 | out_4 | 主输出占比 | 次/主 | 质量 |
|-----------|--------|-------|-------|-------|-------|------------|-------|------|
| 1547.6 | **out_3** | 0.390 | 1.145 | **1.184** | 0.398 | 0.390 | 0.967 | poor |
| 1549.2 | **out_3** | 0.841 | 0.448 | **1.004** | 0.657 | 0.340 | 0.837 | passing |
| 1550.8 | **out_4** | 0.765 | 0.400 | 0.756 | **0.937** | 0.328 | 0.817 | passing |
| 1552.4 | **out_4** | 0.508 | 0.440 | 0.549 | **0.805** | 0.350 | 0.682 | passing |

> 注：这里的功率数值是相对单位（不归一化到 1），所以可以大于 1。每行的数字之和也不等于 1。重要的是各端口之间的相对大小关系。

**最关键的发现**：随着波长从 1547.6 nm 增大到 1552.4 nm，主输出端口从 out_3 变成了 out_4。这证明了 AWG 器件确实表现出**波长依赖性路由**——这是整个实验最核心的物理验证。

**为什么是从 out_3 到 out_4，而不是从 out_1 到 out_4？**

这与输出波导的排列顺序和器件设计有关。更长的波长对应的衍射角更大，在 FPR 输出面的位置更靠外侧。在当前设计中，out_4 是最靠外侧的输出端口，所以最长的波长（1552.4 nm）主要汇聚到 out_4。但 baseline 设计的通道分配还不够精准（从 out_1 到 out_4 应该对应从 ch1 到 ch4），这是优化需要解决的问题。

### 8.6 Stretch Goal 延伸实验

#### 8通道扩展

将输出端口数从4个增加到8个，array pitch 收紧至 1.3 μm，output pitch 为 1.5 μm。

在 1550 nm 处：主输出为 out_7，功率分布：[0.550, 1.081, 0.881, 0.640, 1.045, 0.395, **1.140**, 0.850]

**意义**：验证了 FDTD 工作流程可以扩展到更多通道，不是4通道的特殊情况。

#### 温度依赖性（热光效应）

硅有很强的**热光效应（Thermo-Optic Effect）**：温度升高时，硅的折射率增大（$dn/dT \approx 1.86 \times 10^{-4}$ K⁻¹）。对 AWG 来说，温度变化等效于 n_eff 变化，进而等效于整个传输谱的整体平移。

**测试方法**：在仿真中把温度等效的折射率变化加到 n_eff 上（+40 K 对应 n_eff 增加约 0.0074），然后用 1550.8 nm 光源测试，看主输出端口是否移动。

**结果**：+40 K 时主输出从 out_4（baseline）变成了 out_2。这说明等效相位变化约等于移动了约 2 个通道间距的波长偏移量。

**实际意义**：AWG 对温度非常敏感，实用产品中需要加**温控（TEC）**或设计**无热化（athermal）**结构来抵消温度漂移。

## 9. 优化过程：从粗筛到精调

### 9.1 哪些参数可以优化？为什么？

在 AWG 设计中，ΔL 和衍射级次 m 由通道间距决定，不能随意改变。可以优化的是**几何配置参数**——它们影响光在 FPR 里的汇聚质量：

| 参数 | 物理作用 | 优化方向 |
|------|---------|---------|
| **array pitch（阵列间距）** | 控制 16 个阵列臂出口在 FPR 输入面的总孔径。间距越小，孔径越紧，衍射角越大，光在 FPR 里散布越宽，焦点变宽；间距过大，相邻臂之间可能发生光场耦合 | 寻找"焦点刚好落在输出端口"的间距 |
| **output pitch（输出间距）** | 控制4个输出端口的间距。与阵列孔径/FPR 长度一起决定在 FPR 输出面上相邻通道焦点的间距 | 要与焦点间距匹配，过大则通道漏光多，过小则相邻通道串扰 |
| **output offset（输出偏移）** | 输出端口阵列整体向一侧偏移。用于补偿由于阵列不对称（中心波长不在理论位置）导致的系统性偏差 | 把主焦点对准正确的端口 |
| **FPR length（FPR长度）** | FPR 越长，焦点在输出面的横向间距越大，分辨率越高；但器件也越长 | 粗筛阶段测试，后续固定 |

**三个参数之间的联动关系**：

它们不是独立的。array pitch 决定了相控阵的"有效孔径"（aperture），output pitch 决定了"接收器"的位置，output offset 决定了整体对齐。就像调整一个望远镜——既要调焦距（array pitch），又要把目镜对准正确位置（output pitch），还要微调旋转角（output offset）。

### 9.2 第一步：粗筛（Coarse-to-Fine Screening）

**策略**：在跑任何昂贵的云端仿真之前，先用本地孔径传播近似（Aperture Propagation Approximation）对数百个参数组合做快速打分，找出最有希望的候选点。

本地近似的思路：把 16 个阵列臂出口看作点源，根据给定的波长和参数，计算每个点源到各输出端口的传播相位，叠加后得到每个输出端口的近似强度。这个计算在本地完成，不需要云端，速度极快（几毫秒一次）。

**搜索空间（共 3 × 5 × 5 × 5 = 375 个组合）：**

- FPR 长度：50, 55, 60 μm
- 阵列间距：1.05, 1.10, 1.15, 1.20, 1.25 μm
- 输出间距：1.45, 1.50, 1.55, 1.60, 1.65 μm
- 输出偏移：0.45, 0.50, 0.55, 0.60, 0.65 μm

**粗筛 Top 5 结果：**

| 排名 | 配置 | FPR | array_pitch | output_pitch | offset | 均值主输出占比 |
|------|------|-----|-------------|--------------|--------|----------------|
| 1 | coarse_374 | 60 μm | 1.25 | 1.65 | 0.65 | 0.332 |
| 2 | coarse_249 | 55 μm | 1.25 | 1.65 | 0.65 | 0.332 |
| 3 | coarse_124 | 50 μm | 1.25 | 1.65 | 0.65 | 0.331 |
| 4 | coarse_373 | 60 μm | 1.25 | 1.65 | 0.60 | 0.331 |
| 5 | coarse_248 | 55 μm | 1.25 | 1.65 | 0.60 | 0.331 |

**从粗筛结果中读出的信息**：
- 最优的 array pitch 倾向于 1.25 μm（而不是 baseline 的 1.50 μm），说明更密的阵列孔径有助于改善焦点质量
- FPR 长度对排名影响很小（50/55/60 μm 的名次非常接近），说明这三个长度下焦点质量差不多，后续统一用原始 50 μm
- output pitch = 1.65、offset = 0.65 反复出现，是明确的初始化方向

### 9.3 第二步：局部优化（Local Optimization，以 1550.8 nm 为目标）

基于粗筛结论，在云端 FDTD 上运行 9 个精细配置，全部在 1550.8 nm（中心波长附近）求解：

**完整扫描结果（按 dominant fraction 排序）：**

| 标签 | array_pitch | output_pitch | offset | 主输出占比 | second/first | 质量 |
|------|-------------|--------------|--------|------------|--------------|------|
| **optscan_c** ★ | **1.20** | **1.60** | **0.60** | **0.466** | **0.702** | **general** |
| optscan_b | 1.20 | 1.70 | 0.40 | 0.371 | 0.922 | passing |
| optscan_d | 1.15 | 1.55 | 0.70 | 0.361 | 0.949 | passing |
| optscan_a | 1.30 | 1.80 | 0.30 | 0.358 | 0.829 | passing |
| optscan_h | 1.00 | 1.35 | 1.10 | 0.328 | 0.828 | passing |
| optscan_baseline | 1.50 | 2.00 | 0.00 | 0.328 | 0.817 | passing |
| optscan_e | 1.10 | 1.50 | 0.80 | 0.306 | 0.902 | passing |
| optscan_g | 1.00 | 1.40 | 1.00 | 0.304 | 0.916 | passing |
| optscan_f | 1.05 | 1.45 | 0.90 | 0.284 | 0.969 | **poor** |

**为什么 optscan_c 最优？物理解释：**

- **array pitch 从 1.5 → 1.2 μm**：收紧阵列间距，使出射相控阵的孔径变小。对于固定的 FPR 长度（50 μm），孔径变小意味着在输出面上焦点更集中（衍射展宽减小），主输出端口能接收到更多功率
- **output pitch 从 2.0 → 1.6 μm**：相应地收紧输出端口间距，与焦点间距更好地匹配（焦点间距也因阵列间距减小而变小）
- **output offset = 0.60 μm**：引入了一个正偏移，把输出端口阵列整体向一侧移动，弥补了阵列相位中心与 FPR 几何中心之间的偏差

**一个关键教训——参数要联动调整**：

注意 optscan_f（array pitch=1.05, output pitch=1.45, offset=0.90）反而变成了 poor。这说明参数并不是"越小越好"。当 array pitch 变得很小（< 1.1 μm）时，阵列臂之间的光场开始发生渐逝波耦合（evanescent coupling），能量在臂间泄漏，破坏相位一致性，反而变差。这是一个需要通过仿真才能发现的非线性效应。

**局部最优在全4通道的验证：**

| 波长（nm） | 主输出 | 主输出占比 | second/first | 质量 |
|-----------|--------|------------|--------------|------|
| 1547.6 | out_4 | 0.308 | 0.810 | passing |
| 1549.2 | out_2 | 0.330 | 0.932 | passing |
| **1550.8** | out_2 | **0.466** | **0.702** | **general** |
| 1552.4 | out_1 | 0.370 | 0.896 | passing |

**问题**：中心波长改善了，但边缘通道仍然是 passing。更糟糕的是，主输出端口从 1547.6→1549.2→1550.8→1552.4 nm 分别是 out_4 → out_2 → out_2 → out_1，顺序不单调，说明端口对波长的映射还不规整。

**根本原因**：局部最优是针对单个波长（1550.8 nm）调优的，对其他波长没有施加约束。优化器找到了一个在 1550.8 nm 处最优，但在 1547.6 nm 等处次优的"局部最优"，而不是真正的全局最优。

### 9.4 第三步：全局优化（Global Optimization，兼顾所有4通道）

**优化目标的改变**：

局部优化的目标函数（score）只考虑 1550.8 nm：
$$\text{score}_{\text{local}} = \text{dominant\_frac}_{1550.8} - \text{second/first}_{1550.8}$$

全局优化把目标改成对所有4个通道取平均，同时惩罚最差通道：
$$\text{score}_{\text{global}} = \frac{1}{4}\sum_{i=1}^{4} \text{dom\_frac}(\lambda_i) - \max_i[\text{second/first}(\lambda_i)]$$

（把最大的 second/first 加进惩罚项，以防止某一通道极差）

**新增的扫描配置**（在 seed_local_best 附近做精细格点搜索）：

在 array pitch ∈ {1.20, 1.25}，output pitch ∈ {1.60, 1.65}，offset ∈ {0.55, 0.60, 0.65} 附近搜索。

**全局优化排名（按平均 dominant fraction 排序）：**

| 配置 | array_pitch | output_pitch | offset | 均值主输出占比 | 最差通道占比 | 最大second/first | 综合质量 |
|------|-------------|--------------|--------|----------------|------------|------------------|---------|
| **seed_local_best ★** | **1.20** | **1.60** | **0.60** | **0.368** | **0.308** | **0.932** | **passing** |
| seed_baseline | 1.50 | 2.00 | 0.00 | 0.349 | 0.328 | 0.967 | poor |
| globscan_372 | 1.25 | 1.65 | 0.55 | 0.340 | 0.310 | 0.948 | passing |
| globscan_368 | 1.25 | 1.60 | 0.60 | 0.332 | 0.320 | 0.924 | passing |
| globscan_373 | 1.25 | 1.65 | 0.60 | 0.326 | 0.303 | 0.997 | poor |
| globscan_369 | 1.25 | 1.60 | 0.65 | 0.317 | 0.309 | 0.972 | poor |
| globscan_374 | 1.25 | 1.65 | 0.65 | 0.314 | 0.287 | 0.994 | poor |

**关键发现**：局部最优（seed_local_best，即 optscan_c）在全局优化中仍然排名第一。这说明在这个参数范围内，对中心波长最好的配置，也恰好对全通带平均最好。

**但这并不意味着优化已经完成**：全局最优的平均 dominant fraction 只有 0.368，全部4通道仍处于 passing 水平。要真正达到 general（均值 ≥ 0.40），需要更大范围的参数搜索，甚至可能需要引入全新的几何自由度（如 FPR 形状优化、Rowland 圆几何）。

### 9.5 前三轮优化结论总结

前三轮优化（粗筛 → 局部优化 → 全局优化）均固定了 N=16 根阵列臂和 FPR=55 μm：

```
最终最优配置（局部最优 = 全局最优）：
  array pitch   = 1.20 μm  （初始 1.50 μm，缩紧 20%）
  output pitch  = 1.60 μm  （初始 2.00 μm，缩紧 20%）
  output offset = 0.60 μm  （初始 0.00 μm，加入正偏移）

性能变化：
  中心波长 1550.8 nm：passing (0.328) → general (0.466)  ↑ +42%
  全4通道平均：       passing (0.349) → passing (0.368)  ↑ +5%
  最差 second/first： 0.967           → 0.932             ↓ 改善

瓶颈诊断：
  ✓ 找到了一组比 baseline 有意义提升的几何配置
  ✓ 证明了通过参数调整可以有效改善器件性能
  ✗ 整个4通道系统尚未整体达到 general 水平
  ✗ 主输出端口对波长的映射仍不规整（ch1→out4, ch2→out2, ch3→out2, ch4→out1）
  → 根本原因：N=16 时 array factor 3dB 峰宽 ≈ FSR/16 = 1.2 nm，
    与通道间距 1.6 nm 太接近，相邻通道存在严重的能量交叠
```

### 9.6 第四步：改进设计——增加阵列臂数量（N=24）

#### 核心想法的来源

前三轮优化只调整了几何参数（间距和偏移），但从来没有改变最根本的结构参数——**阵列臂数量 N**。

理论分析表明，AWG 传输谱的主峰宽度（3dB带宽）正比于 FSR/N：

$$\text{3dB BW} \approx \frac{\text{FSR}}{N} = \frac{19.2 \text{ nm}}{N}$$

| N | 3dB 峰宽 | 与通道间距（1.6 nm）的比值 | 相邻通道重叠程度 |
|---|---------|------------------------|----------------|
| 16 | 1.20 nm | 0.75（峰宽是间距的 75%） | **严重重叠** |
| 24 | 0.80 nm | 0.50（峰宽是间距的 50%） | 中等重叠 |
| 32 | 0.60 nm | 0.375 | 轻度重叠 |

**大白话**：N=16 就像 16 支步枪排成一排同时扫射，打出的"光束"太宽，打中这个靶子的时候旁边的靶子也被打到了。增加到 N=24 就是加到 24 支步枪，光束变窄，相邻通道的干扰减少。

#### 实验配置（4组 × 4波长 = 16次 FDTD，共消耗 ~1.66 FlexCredit）

| 标签 | N | FPR（μm） | array_pitch | output_pitch | offset | 改进意图 |
|------|---|-----------|-------------|--------------|--------|---------|
| impr_A | 24 | 55 | 1.20 | 1.60 | 0.60 | 只增加臂数，其他不变，检验 N 单独的贡献 |
| impr_B | 24 | 55 | 1.20 | 1.60 | 0.90 | N=24 + 增大偏移 |
| **impr_C** ⭐ | **24** | **55** | **1.20** | **1.80** | **0.90** | **N=24 + 更宽 output_pitch** |
| impr_D | 24 | 70 | 1.20 | 2.00 | 0.80 | N=24 + 更长 FPR（通道焦点间距增大） |

#### 完整实验结果

**Config A（N=24，同旧最优几何参数）：**

| 波长（nm） | 主输出 | dom_frac | s2f | 质量 |
|-----------|--------|----------|-----|------|
| 1547.6 | out_1 | 0.313 | 0.846 | passing |
| 1549.2 | out_2 | 0.393 | 0.822 | passing |
| 1550.8 | out_2 | 0.405 | 0.977 | poor |
| 1552.4 | out_2 | 0.395 | 0.635 | passing |

平均 dom_frac = 0.377，比 N=16 的 0.368 略有提升，但 1550.8 nm 反而变差（s2f=0.977 → poor）。说明仅仅增加臂数而不调整 output_pitch，不能改善所有通道。

**Config B（N=24，offset=0.9）：**

| 波长（nm） | 主输出 | dom_frac | s2f | 质量 |
|-----------|--------|----------|-----|------|
| 1547.6 | out_4 | 0.300 | 0.908 | poor |
| 1549.2 | out_4 | 0.309 | 0.966 | poor |
| 1550.8 | out_2 | 0.327 | 0.866 | passing |
| 1552.4 | out_2 | 0.377 | 0.906 | passing |

平均 dom_frac = 0.328，反而变差。说明 offset 调大之后，出口位置偏离了焦点位置。

**平均 dom_frac = 0.403**（项目迄今最高），**2个通道达到 general**。output_pitch 从 1.6→1.8 μm 给各通道焦点留出了更多横向空间，与 N=24 的更窄峰宽配合产生了协同效应。

**Config D（N=24，FPR=70 μm，更长自由传播区）：**

#### 四轮优化全程性能演变

| 阶段 | 配置 | 平均 dom_frac | 达到 general 通道数 | max s2f |
|------|------|--------------|--------------------|---------| 
| Baseline | N=16，1.5/2.0/0.0 | 0.349 | 0 | 0.967 |
| 局部优化 | N=16，1.2/1.6/0.6 | 0.368 | 1（仅中心波长） | 0.932 |
| 全局优化 | N=16，1.2/1.6/0.6 | 0.368 | 1 | 0.932 |
| **本轮最优（impr_C）** | **N=24，1.2/1.8/0.9** | **0.403** | **2** | **0.924** |

**相比项目 Baseline 的总体改善：**
- 平均 dom_frac：0.349 → **0.403**（+15.5%）
- 达到 general 的通道：0 → **2 个**
- max s2f：0.967 → **0.924**（串扰改善）

#### 为什么 1550.8 nm 在新配置下反而偏弱？

这是 AWG 器件设计中的典型"中心通道异常"现象：

在当前的矩形 FPR 设计（非 Rowland 圆）中，中心波长（λ₀=1550 nm）的焦点正好落在 FPR 几何中心附近。当 output_pitch 增大（1.6→1.8 μm）且 offset 增大（0.6→0.9 μm）时，输出端口阵列整体向正方向偏移，中心焦点位置反而没有输出端口对准，而边缘通道的焦点恰好落在某个端口上。

这再次说明了**Rowland 圆几何**的必要性：只有在 Rowland 圆上排列端口，才能保证所有通道（包括中心通道）都能在正确位置找到匹配的输出端口，不会出现这种"顾此失彼"的现象。

## 11. 结论与局限

### 11.1 实验成功了什么？

1. **完成了完整的 AWG 设计链**：从 MODE 求解 n_g，到推导 ΔL 和 m，到阵列因子频谱模型，到 FDTD 器件验证，每一步都有可追溯的数据和逻辑
2. **证实了波长依赖性路由**：云端 FDTD 清楚地显示主输出端口随波长移动，这是真实麦克斯韦方程组的求解结果，不是数学模型的假设
3. **通过三轮几何参数优化提升了性能**：中心波长处 dominant fraction 从 0.328 提升到 0.466（+42%），质量从 passing 升至 general
4. **通过改变阵列臂数量（N=16→24）实现了更大突破**：全4通道平均 dominant fraction 从 0.349 提升到 **0.403**（+15.5%），达到 general 的通道从 1 个增加到 **2 个**。这是整个优化周期中最有效的单一改变
5. **发现了核心瓶颈**：N=16 时 array factor 峰宽（1.2 nm）与通道间距（1.6 nm）过于接近，是限制分离度的根本物理原因
6. **Stretch goals 初步验证**：8通道、温度依赖（热光效应）、串扰改善三个方向都有了独立 FDTD 结果

### 11.2 当前的局限和为什么

| 局限 | 深层原因 |
|------|---------|
| 全4通道尚未整体达到 general | 矩形 FPR 几何（非 Rowland 圆）导致中心通道与边缘通道的聚焦质量不一致；N=24 改善了边缘通道，反而使中心通道偏离 |
| 主输出端口映射不规整 | 缺少 Rowland 圆几何：标准 AWG 把输入/输出端口排在 FPR 的 Rowland 圆上，当前直线排列导致各通道焦点位置不均匀 |
| 简化模型4通道结果完全相同 | 本地阵列因子模型是对称的，无法预测边缘通道与中心通道之间的性能差异 |
| 未获得真实插损/串扰数值 | 需要完整 3D FDTD（约 7.8 FlexCredit/次），超出课程预算；2D 有效折射率方法忽略了弯曲损耗和厚度方向的模式泄漏 |

### 11.3 若继续做，优先级排序（已更新）

**已验证有效、可立即实施的改进方向：**

1. **N=24 + 继续细化 output_pitch/offset**（最低成本）：本次实验证明 N=24 是有效的。在 impr_C（pitch=1.8, offset=0.9）基础上，对 offset 做 ±0.1 μm 的精细扫描，可能进一步改善 1550.8 nm 通道，同时保留其他通道的 general 质量。预算约 0.3 FlexCredit
2. **N=32 + 更长 FPR（90 μm）**：峰宽降至 0.6 nm，分离度进一步提高；需要适当增大 SLAB_SPAN_Y_UM（约 44 μm）。预算约 0.5 FlexCredit

**需要更大改动但效果更根本的方向：**

3. **Rowland 圆 FPR 几何**：把输入/输出端口从直线改为排列在 Rowland 圆上（半径 = FPR长度的一半），这是消除"中心通道偏弱"问题的根本解法。需要重写 `build_simulation` 中的几何设置
4. **完整 3D FDTD**：性能评估金标准，但约 7.8 FlexCredit/次，不在当前预算内
5. **Apodization（渐变加权）**：对不同臂的耦合效率做非均匀加权，可以压制旁瓣，把串扰从约 -14 dB 推到 -30 dB 以下

| 英文术语 | 中文翻译 | 一句话解释 |
|----------|---------|-----------|
| AWG (Arrayed Waveguide Grating) | 阵列波导光栅 | 利用多臂相控干涉实现波长分光的芯片级器件 |
| WDM | 波分复用 | 把多路不同颜色的光信号叠加在一根光纤里传输 |
| C-band | C波段 | 光通信主力波段，1530–1565 nm |
| MODE solver | 模式求解器 | 求解波导横截面中麦克斯韦方程的本征值问题 |
| FDTD | 时域有限差分法 | 数值方法：在时间上逐步演化电磁场，最后 FFT 得频域结果 |
| n_eff | 有效折射率 | 光在波导里"感受到"的等效折射率，由截面几何和材料决定 |
| n_group | 群折射率 | 决定光脉冲传播速度的折射率，= n_eff - λ·dn_eff/dλ |
| Propagation Constant β | 传播常数 | 光沿波导传播时每单位长度积累的相位，= 2π·n_eff/λ |
| FSR | 自由光谱范围 | AWG 传输谱的周期；超出 FSR 的光会产生混叠 |

| Diffraction Order m | 衍射级次 | 相邻臂之间相位差是 2π 的几倍；m 越大 FSR 越小 |
| ΔL | 路径长度差 | 相邻阵列臂的长度差，AWG 的核心设计参数 |
| Array Factor | 阵列因子 | N 根等间距相位递增的辐射源叠加形成的干涉图样 |
| Insertion Loss | 插入损耗 | 光通过器件后损失的功率，单位 dB，越小越好 |
| Crosstalk | 串扰 | 邻道信号混入目标通道的程度，单位 dB，越负越好 |
| Dominant Fraction | 主输出占比 | 总功率中最强输出端口的占比，越大越好（理想值 1.0）|
| Second/First | 次强/最强比 | 第二强输出 / 最强输出，越小越好（理想值 0）|
| Array Pitch | 阵列间距 | 阵列臂在 FPR 接口处的中心间距 |
| Output Pitch | 输出间距 | 输出波导的中心间距 |
| Output Offset | 输出偏移 | 输出端口阵列相对 FPR 中心的整体横向偏移 |
| Rowland Circle | Rowland 圆 | AWG 标准几何：输入/输出波导排在 FPR 的 Rowland 圆上，使聚焦像差最小 |
| Apodization | 渐变加权 | 对不同阵列臂的耦合系数做非均匀加权，以降低传输谱的旁瓣 |
| Thermo-Optic Effect | 热光效应 | 温度变化引起折射率变化（Si：dn/dT ≈ 1.86×10⁻⁴ K⁻¹）|
| SOI (Silicon-on-Insulator) | 绝缘体上硅 | 主流硅光子工艺平台，器件层厚度通常 220 nm |
| FlexCredit | 计算积分 | Tidy3D 的计算资源单位 |
| 2D Effective Index Method | 2D 有效折射率方法 | 用 n_eff 替代垂直方向，把 3D 问题降维为 2D |
| TE-like mode | 类TE模 | 电场主要方向平行于芯片平面的导模 |
